## 5. Community Health Heatmap System *(PRD §6.1)*

### 5.1 Map & Visualization
- [ ] Integrate Google Maps Platform
- [ ] Display village-level health heatmap within configurable radius (default 30 km)
- [ ] Aggregate anonymised data from consultations, symptoms, ASHA reports, pharmacy logs

### 5.2 Alert System
- [ ] Implement 3-tier color-coded zones:
  - [ ] **GREEN** (Safe): <5% weekly symptom increase vs 4-week baseline
  - [ ] **YELLOW** (Early Warning): 5–20% increase OR 3+ similar symptoms in 48h → alert to PHC + ASHA
  - [ ] **RED** (Outbreak): >20% increase OR 5+ identical symptoms in 72h → auto DHO notification + resource alert

### 5.3 Filtering & Real-Time Updates
- [ ] Filter by: disease/symptom type, time range (24h/7d/30d/custom), geographic radius
- [ ] Near-real-time updates every 15 min via WebSocket push
- [ ] Cached snapshot for low-bandwidth users

### 5.4 Cluster Detection
- [ ] Detect clusters: fever, respiratory, diarrhoea, skin infections, vector-borne diseases
- [ ] Auto-alert DHO on red zones via SMS + dashboard push

### 5.5 Low-Bandwidth Mode
- [ ] Serve static PNG heatmap snapshot when connection < 2G
- [ ] Show "Last updated: X hours ago" timestamp for offline cache

### 5.6 Data Privacy *(PRD §6.1.2)*
- [ ] Aggregate at village level only — no individual patient location/identity
- [ ] Minimum 5 cases threshold before village appears on map (k-anonymity)
- [ ] Data retention: 2 years source records, indefinite aggregated trends

### 5.7 Performance *(PRD §6.1.3)*
- [ ] Heatmap loads within 3s on 3G
- [ ] Handle 10,000 concurrent viewers during outbreak events

## 9. AI Triage System *(PRD §6.5)*

### 9.1 Stage 1 — Emergency Screening *(PRD §6.5.1)*
- [ ] 3 questions, 30-second completion target
- [ ] Q1.1: Breathing difficulty / chest pain → CRITICAL
- [ ] Q1.2: Lost consciousness / limb immobility → CRITICAL
- [ ] Q1.3: Uncontrolled bleeding / severe injury → CRITICAL + ambulance prompt

### 9.2 Stage 2 — Severity Assessment *(PRD §6.5.1)*
- [ ] 8–12 dynamically selected questions, 3–5 min
- [ ] Domains: pain intensity (0.25), duration (0.15), progression (0.20), associated symptoms (0.15), chronic conditions (0.15), age/vulnerability (0.10)

### 9.3 Stage 3 — Context & History *(PRD §6.5.1)*
- [ ] 3–5 questions, 1–2 min
- [ ] Covers: travel, contact with sick individuals, current meds, allergies, visit reason

### 9.4 Severity Scoring Algorithm *(PRD §6.5.2)*
- [ ] Weighted composite score (0–10) from Stage 2 domain scores
- [ ] Vulnerability multipliers: >60 yrs (+10%), <5 yrs (+15%), pregnancy (+20%)
- [ ] Chronic disease penalties: diabetes (+0.5), heart disease (+1.0), immunocompromised (+1.5)
- [ ] NLP severity from voice input integration
- [ ] Temporal urgency: worsening rapidly → +1.0
- [ ] **CRITICAL** (8–10): immediate consult + doctor alert
- [ ] **URGENT** (5–7): <15 min wait, next in queue
- [ ] **MODERATE** (3–4): <45 min wait
- [ ] **ROUTINE** (1–2): <90 min wait

### 9.5 Dynamic Re-Prioritisation *(PRD §6.5.3)*
- [ ] Triggers: patient symptom update, ASHA flag, doctor manual elevate/lower
- [ ] Auto re-scoring every 5 min for all queued patients
- [ ] +2 point increase → immediate queue promotion + doctor notification
- [ ] -2 point decrease → voluntary move down (patient SMS consent)
- [ ] Full audit trail logging

## 17. Accessibility *(DD §7 + PRD §12)*

- [ ] Body text on white: ≥4.5:1 (WCAG AA)
- [ ] Large text (18px+ Regular / 14px+ Bold): ≥3:1
- [ ] Never convey meaning through color alone — pair with icon/text

### 17.2 Touch Targets *(DD §7.2)*
- [ ] Minimum 44×44px on all interactive elements
- [ ] Invisible padding on small icons for larger tap area
- [ ] Full-height tap zone on bottom nav tabs

### 17.3 Screen Reader *(DD §7.3)*
- [ ] All images/illustrations: descriptive `accessibilityLabel`
- [ ] Interactive elements: `accessibilityRole` (button, tab, etc.)
- [ ] Dynamic content: `accessibilityLiveRegion`
- [ ] Focus order: top→bottom, left→right

### 17.4 Text Sizing *(DD §7.4)*
- [ ] Dynamic Type / font scaling support
- [ ] Minimum production font: 11px (captions only)
- [ ] Proportional line height scaling
- [ ] Card reflow up to 200% text scale

### 17.5 WCAG *(PRD §12)*
- [ ] WCAG 2.1 Level AA compliance for all web interfaces

## 18. Non-Functional Requirements *(PRD §12)*

- [ ] API response p95: <500ms (non-AI endpoints)
- [ ] AI triage latency: <3s end-to-end
- [ ] NLP pipeline latency: <2s for 30-sec audio
- [ ] Heatmap load on 3G: <3s
- [ ] System uptime SLA: 99.5% monthly (<3.6 hrs downtime/month)
- [ ] Concurrent active users: 50,000 (500 PHCs target)
- [ ] PWA offline: core features (queue check, record view) work offline
- [ ] Localization: 10 languages at launch (Hindi + 9 regional)
- [ ] Device support: Android 5.0+, iOS 12+, any modern browser
- [ ] SMS delivery: <30s for appointment confirmations
